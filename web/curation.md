# Curation

1. Review an Inbox candidate for safety, scope, source, confidence, and conflicts.
2. Consolidate only confirmed, non-conflicting updates into Core or the correct area record.
3. Add a Core `CHANGELOG.md` entry naming date, candidate source, target, result, and reason.
4. Move the processed candidate to Archive with its status updated to processed, held, or rejected.

Temporary material is excluded. Conflicting preferences, goals, decisions, and local/remote disagreements are held for explicit user approval. A rejected candidate is archived with a safe reason; never copy secrets, credentials, raw chats, or CLI technical state into canonical Web memory.
