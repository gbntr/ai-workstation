# Session Closure

For a relevant session, separate durable items from temporary notes. Create at most one pending candidate using [the template](../templates/session-memory-candidate.md). Include source and date, proposed destinations, exclusions, and any conflict. Do not update canonical memory silently.

With an explicitly confirmed available app write, save the candidate to Inbox. Otherwise present the Markdown for the user to save manually to the synchronized local clone. State that discovery may lag until synchronization completes.
